# 项目理解

---

## 一、项目目标

| 层次 | 目标 | 学习成果 |
|:---:|------|----------|
| **第一层：功能实现** | 学习实现具体功能所需的技术技能 | 前后端编写、LLM 接口调用、Mattermost 命令配置等具体技术 |
| **第二层：链路理解** | 理解数据在各服务之间的流转路径 | 建立系统思维，将 Mattermost、后端、数据库、AI 间数据传输理清 |
| **第三层：协作规范** | 熟悉团队协作流程与工程规范 | 掌握 Git 分支管理、PR提交流程、根据Code Review修改等规范 |
| **第四层：心智模型** | 形成可迁移的项目开发方法论 | 建立 **"理解需求 → 拆解模块 → 分步实现 → 集成联调"** 的思考框架 |

---

## 二、整体链路

---

### 第一阶段：分步学习

各基础作业对应最终完整系统的不同模块，旨在让成员在可控范围内逐项掌握技能：

| 作业 | 对应模块 | 培养能力 |
|:---:|----------|----------|
| **作业1（Git 协作）** | 版本控制与协作 | 代码提交、分支管理、PR 流程 |
| **作业2（Mattermost 环境）** | 第三方服务集成 | 环境配置、文档阅读、依赖管理 |
| **作业3（Task API）** | 后端服务 | API 设计、请求响应处理 |
| **作业4（数据库）** | 数据持久层 | 表结构设计、关系建模、CRUD 操作 |
| **作业5（Agent）** | AI 能力集成 | Prompt 设计、LLM API 调用、结构化输出处理 |
| **作业6（前端）** | 用户界面 | API 调用、数据渲染、页面展示 |
| **作业7（Docker）** | 部署与运维 | 容器化配置、多服务编排、环境标准化 |

---

### 第二阶段：集成整合

在完成各基础作业后，最终大作业要求将各模块组装为一个 **完整可运行的系统**。以 Mattermost 为交互入口，串联后端 API、数据库、AI Agent 和前端页面，形成从 **用户输入到结果输出** 的端到端功能闭环。

---

### 第三阶段：独立应用

完成大作业后，成员应当具备 **独立承担开发任务** 的能力。面对新的功能需求时，能够按照统一的方法论完成：

1. **需求分析** — 理解需求背景与目标
2. **模块拆解** — 将需求拆分为可独立实现的子模块
3. **分步实现** — 逐个模块进行开发实现
4. **集成联调** — 将各模块整合为完整功能
5. **代码提交** — 按照规范完成代码提交流程

将学习成果迁移至后续实际项目开发中。

## 三、当前理解

### 1. 具体链路

#### app.py 后端主服务

- 教师在 Mattermost 上面发布作业，作业内容通过后端存入数据库
- 学生提交作业，通过后端存入数据库

#### agent.py AI 评估

- 通过 `app.py` 读取数据库内容，整合作业要求、提交内容、评分标准等进 Prompt
- 调用 AI API，返回输出结果到 `app.py`，写回数据库

#### db.py 数据库操作

- 负责数据库具体执行操作，供 `app.py` 调用

#### 前端界面

- 通过 `app.py` 获取数据库中 AI 返回的评估结果，进行展示，并供修改
- 确定后数据返回数据库
- 前端通过后端调用 Mattermost API，返回结果给学生

### 2. 部署方式

| 项目 | 说明 |
|:----:|------|
| **文件组织** | 所有代码和配置文件放在同一个本地文件夹下 |
| **运行方式** | 在本地使用 **Docker 编排** 运行所有服务 |
| **访问入口** | 通过 `localhost` 访问 |
| **版本控制** | 文件同时上传至 **GitHub** 仓库 |
