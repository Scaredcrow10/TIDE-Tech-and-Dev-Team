# Understanding - KejieXU
###### 2026.06.25

### 一、项目核心背景与目标
当前任务旨在搭建一套基于多智能体（multi-agent）架构的项目组织管理系统，解决小型厂商/组织团体因缺乏标准化管理工具导致的问题：
- 成员沟通成本高
- 风险无法自动监督
- 项目进度无专人跟进
- 财务流程混乱

**核心价值**：以低成本为组织提供精细化、自动化的团队管理解决方案。

### 二、核心业务流程
#### 1. 用户/成员操作行为
| 角色          | 核心操作                                                                 |
|---------------|--------------------------------------------------------------------------|
| 高层/业务负责人 | 发起新项目立项：上传需求文档、设定目标工期/预算、指定所需人员、明确业务目标 |
| 项目经理/员工  | 提交日常事务：任务变更、进度更新、请假、报销、资源调配申请、会议纪要素材   |
| HR/财务       | 录入组织基础静态数据：人员档案、财务核心数据                             |
| 全体成员       | 通过对话、表单、文件上传、第三方工具（如飞书）同步录入零散业务信息         |

#### 2. 系统多智能体设计
系统内置多角色智能体（agent），并构建层级协作关系：
- **职能型智能体**：HR agent（人力信息管理）、财务agent（财务事务管理）等；
- **统筹型智能体**：高管agent（全局信息统筹、接收用户请求、输出最终结果）；
- **核心关键**：搭建多智能体体系，明确agent间的协作与层级关系。

#### 3. 系统输出能力
- 标准化产出：任务清单、各类报表、可视化看板、复盘文档等业务成果；
- 个性化能力：根据用户请求输出定制化回答，支持智能体架构自优化。

### 三、MVP范围与优先级
#### 1. MVP必须包含能力
- 基础组织架构与人员权限管理
- 核心调度、项目规划、进度监控类核心智能体
- 单项目立项流程
- AI 任务拆解、人员自动分配
- 进度更新、简易延期风险预警
- 基础消息通知提醒
- 单项目进度看板、自动生成简易周报

#### 2. MVP暂不包含能力
- 完整财务/人事/合规类智能体
- 多项目全局资源调度与冲突处理
- 复杂多级审批流程
- 深度第三方系统对接
- 多模态智能解析
- 高级数据预测分析
- 知识库迭代优化、多租户支持等高级拓展能力

#### 3. 最高优先级任务
1. 搭建基础多智能体协作核心链路；
2. 打通单项目“立项-进度管控”完整基础闭环；
3. 搭建基础组织权限体系；
4. 快速落地核心AI项目管理能力。

### 五、开发协作规范
代码管理流程：
1. git clone 拉取远程仓库；
2. 切换至main主干并更新，保证代码最新；
3. 新建个人独立业务分支，隔离开发；
4. git add 精准提交文件至暂存区；
5. commit 提交本地版本（规范备注）；
6. push 个人分支至远程仓库；
7. 通过GitHub提交PR申请合并至主干。
