## 1. 我理解的项目目标

> 我理解这个项目当前的目标是，实现一个智能的项目管理平台，管理者能够在该平台上创建并发布任务，其他人能够接收任务并提交反馈，支持多种应用场景。操作支持自然语言进行，一步到位，摒弃繁琐的操作。系统能够实现基础的各类需求，包括但不限于前后端通信，命令行和简单的图形界面，数据库管理等，并接入外部的大语言模型，通过自然语言输入自动生结构化任务列表。

## 2. 我理解的整体链路

- 若以最终验收大作业为准，用户就是学生与教师，教师能够通过平台发布作业，学生能够通过平台接收并提交作业，平台会自动生成学生作业的初步批改，供教师审阅。
- 系统主要处理作业的写入与存档，数据库通过PostgreSQL来构建。处理不同用户间的通信，通过Mattermost来进行。自动批改学生上传的作业，调用外部LLM。
- 最终实现一个智能易用的作业发布与批改教务平台。

## 3. 我理解的 MVP 范围

- MVP 应该包含：简单图形化操作界面，数据库，本地模拟通信，接入大语言模型进行自动批改汇总功能。
- MVP 暂时不需要包含：远端通信，用户权限，日志记录等。
- 我认为优先级最高的是：数据库，自动批改汇总功能。
